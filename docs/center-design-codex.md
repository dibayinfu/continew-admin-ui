# 数字大屏指挥中心整体优化升级设计文档（Codex 版）

> 菜单：7月份阶段 - 数字大屏指挥中心  
> 目标页面：`src/views/sanitation/command-center-july.vue`  
> 数据文件：`src/views/sanitation/data/command-center-july-mock.ts`  
> 目标分辨率：4810 x 1568  
> 文档日期：2026-06-25

## 结论

本次升级不建议把大屏继续做成“左侧指标 + 中间地图 + 右侧 Tab”的常规后台看板。该页面同时承担两类职责：

1. 日常运营调度：调度员需要看到车辆、箱体、告警、任务、地图态势，并能快速定位和处置。
2. 政务参观展示：客户会带政府官员参观，大屏必须快速展示项目体量、平台处理能力、累计数据成果和运行效率。

因此新版大屏的主线应调整为：

**地图态势作为第一主画面，承载车辆、箱体、收集点、任务、告警和视频联动；项目规模和累计成果收敛到左侧综合总览，作为参观展示的稳定信息底座。**

DeepSeek 版方案方向是对的，尤其是“双重定位”和“模块二加入基础档案”的判断。但它仍偏模块罗列，缺少以下落地约束：

- 4810 宽屏下，不能继续用窄列 Tab 堆叠关键信息，应把右侧任务、告警、资产分区并列展开。
- 不建议保留过高的全宽模块二。模块二占用纵向空间后会压缩地图，而地图才是调度和参观时最能解释业务链路的主画面。
- “覆盖人口”不应使用未经确认的 32.6 万。结合现有项目文档，建议先按 **约 19 万人** 展示，后续以甲方最终档案数据为准。
- 模拟数据规模需要与展示规模区分：真实展示口径是 8 个乡镇、157 个村庄、370 个箱子、500 辆小三轮、20 台小勾臂、6 台大勾臂，页面 mock 可抽样渲染，但模块三档案必须按全量口径展示。

## 本轮调整结论

针对“模块四地图过小、模块二过大”的问题，本版文档调整为：

1. **取消全宽大面积模块二**：基础档案和累计成果不再单独占 190px 高度的横向大带。
2. **模块三升级为“项目与运营总览”**：把静态数据和动态累计数据放在左侧第一列，形成可读、稳定、不会遮挡地图的综合指标区。
3. **今日/当前数据进入地图浮层**：车辆在线率、今日任务、星标告警、今日垃圾量、箱体异常等指标，以地图顶部/底部浮层、图层统计、角标形式叠加在模块四中。
4. **模块四显著放大**：地图建议跨中间主区域，宽度从约 2200px 提升到约 3000px，高度吃满主体区，点击车辆后可展示详情卡和视频，不再被过多顶部指标压缩。

## 设计目标

### 业务目标

- 展示龙安区乡镇生活垃圾收运项目的全域覆盖能力。
- 展示“村庄垃圾产生 - 小三轮收集 - 小勾臂箱暂存 - 小勾臂车转运 - 大勾臂箱中转 - 大勾臂车转运 - 焚烧厂处置 - 数据统计与对账”的完整链路。
- 支持调度员日常查看告警、任务、箱体、车辆状态。
- 支持政府参观时快速理解项目规模、平台效率、累计成果和数字化监管能力。

### 视觉目标

- 第一眼看到全域地图态势，明确车辆、箱体、收集点、中转站、焚烧厂和告警分布。
- 第二眼看到项目规模和累计成果，知道平台管了多少、处理了多少。
- 第三眼看到调度闭环能力，能从告警、任务、车辆详情、视频联动进入处置。
- 宽屏横向展开，地图优先，指标和图表围绕地图服务，不反客为主。

## 模块二：项目档案与成果数据

### 定位调整

模块二不再作为全宽独立大模块出现，而是作为模块三“项目与运营总览”的数据来源。原因：

- 地图需要承载车辆、箱体、收集点、任务、告警和视频入口，是页面第一主画面。
- 静态档案和累计成果属于认知型信息，重要但不需要占用整屏横向高度。
- 今日/当前运行数据更适合贴近地图，以图层浮层方式展示，让调度员看到指标变化时能马上关联空间对象。

## 模块四：全域态势地图

### 定位

地图是运营调度主工作区，也是参观展示时最重要的业务解释画面。它不只是底图，而是承载车辆、箱体、收集点、中转站、焚烧厂、告警、任务线路、车辆视频入口的核心模块。

新版布局中，模块四应显著放大：

- 宽度建议约 3000px，占主体区域 60% 以上。
- 高度吃满 Header 与底部操作栏之间的主体区，不再被全宽模块二压缩。
- 地图内部允许放置轻量浮层，但浮层不能遮挡主要点位和线路。
- 车辆点击后的详情卡和视频入口在地图内侧抽屉展示，不跳出地图主上下文。

### 为什么地图必须放大

| 原因 | 说明 |
|---|---|
| 信息密度最高 | 车辆、箱体、收集点、中转站、焚烧厂、告警和任务都需要上图 |
| 调度动作依赖地图 | 告警定位、车辆定位、任务轨迹、箱体满溢都需要空间判断 |
| 参观展示更直观 | 政府参观时，地图比指标更容易解释“覆盖哪里、车在哪、箱在哪、怎么运转” |
| 视频联动需要空间上下文 | 点击车辆后查看车辆详情和视频，应仍能看到车辆位置、附近任务和线路 |

### 图层

| 图层 | 内容 | 样式 |
|---|---|---|
| 行政边界 | 8 个乡镇/街道 | 淡色边界，参观模式可分色 |
| 村庄/收集点 | 157 村庄、重点收集点 | 蓝/青色点 |
| 箱体 | 小勾臂箱、大勾臂箱 | 绿正常、黄预警、红满溢、灰离线 |
| 车辆 | 小三轮、小勾臂车、大勾臂车 | 方向箭头，按状态着色 |
| 中转站 | 移动压缩中转站 | 方形站点，显示负荷 |
| 焚烧厂 | 末端处置点 | 固定高亮标识 |
| 告警 | 主动安全、满溢、低电量、离线 | 红/橙闪烁 |
| 任务 | 收运中任务线路 | 虚线轨迹 |

### 地图浮层

今日/当前指标从模块三迁移到地图浮层，建议分布如下：

| 浮层位置 | 内容 | 说明 |
|---|---|---|
| 左上角 | 车辆在线率、在线/离线/充电数量 | 与车辆图层绑定 |
| 顶部中间 | 今日任务、收运中、已完成、已超时 | 点击筛选任务线路 |
| 右上角 | 待处理告警、严重告警、今日新增 | 点击筛选告警点 |
| 左下角 | 今日垃圾量、完成率、SLA 达标率 | 作为当日运行摘要 |
| 右下角 | 图层开关：车辆、箱体、收集点、中转站、告警、任务 | 每个图层带数量角标 |
| 中转站点位旁 | 当前负荷环形进度 | 超 80% 变黄，超 90% 变红 |

### 交互

- 点击车辆：地图右侧弹出车辆详情抽屉，显示车牌、车型、驾驶员、速度、载重、在线状态、今日里程、今日趟次、告警数、轨迹回放、实时视频入口。
- 点击箱体：显示编号、乡镇、村庄、满溢率、电量、温度、锁状态、最近上报。
- 点击告警：显示等级、触发时间、位置、处理状态，可创建任务。
- 点击任务：地图高亮起点、终点、车辆当前位置和任务轨迹。
- 图层开关：车辆、箱体、站点、告警、任务。

### 车辆详情与视频

点击车辆后的详情不建议做全屏 Modal，建议做地图内右侧抽屉：

| 区域 | 内容 |
|---|---|
| 车辆基础信息 | 车牌、车型、驾驶员、联系方式、所属乡镇 |
| 实时状态 | 速度、方向、载重、在线状态、最后上报 |
| 作业数据 | 今日里程、今日趟次、当前任务、预计到达时间 |
| 安全数据 | 今日告警数、最近告警类型、处理状态 |
| 轨迹 | 最近 30 个点、播放/暂停、速度选择 |
| 视频 | 前摄像头、后摄像头、车内摄像头入口或缩略窗 |

视频展示建议：

- 默认在车辆详情抽屉内展示 1 个小预览窗，避免遮挡地图。
- 点击“查看实时视频”后打开地图内悬浮窗，支持前/后摄像头切换。
- 视频窗口保留车辆位置小地图或“定位车辆”按钮，避免用户脱离空间上下文。

### 参观模式

参观模式下地图默认开启全图层，但弱化单个告警闪烁，强调覆盖范围、车辆在线、业务链路流动效果。地图应成为参观讲解的主背景，而不是被顶部指标或侧边表格挤压。

## 模块五：实时告警与处置能力

### 定位

体现平台“能发现问题、能定位问题、能推动处理”。

### 展示结构

顶部三项统计：

- 待处理告警
- 严重告警
- 今日新增

列表分两组：

1. 主动安全：疲劳驾驶、分神驾驶、接打电话、抽烟、车道偏离、跟车过近、行人碰撞预警。
2. 箱体设备：满溢、低电量、离线、高温。

### 操作

- 点击告警定位地图。
- 告警详情支持“已查看”“已处理”“创建任务”。

## 验收标准

### 展示验收

- 4810 x 1568 下无横向拥挤、无文字重叠、无核心指标隐藏。
- 地图宽度占主体区 60% 以上，车辆、箱体、收集点、中转站、告警和任务线路可清晰辨识。
- 首屏直接看到 8、157、370、500、20、6、覆盖人口，但这些信息不压缩地图。
- 参观模式下 5 秒内能通过地图理解覆盖范围和业务链路，通过左侧总览理解项目规模和成果。
- 调度模式下 5 秒内能在地图浮层看到待处理告警、今日任务、车辆在线和地图异常。

### 业务验收

- 告警点击能联动地图。
- 任务点击能显示任务详情和轨迹。
- 箱体点击能看到满溢率、电量、温度、状态。
- 车辆点击能在地图内抽屉看到驾驶员、速度、轨迹、视频入口。
- 满溢告警能创建清运任务。

### 技术验收

- 正式分辨率 `mode-formal` 为 4810 x 1568。
- 测试分辨率 `mode-test` 可在 1280 以上正常预览。
- 图表 resize 后不溢出。
- mock 全量统计和地图抽样数据口径分离。
- 新增样式不影响其他卫生模块页面。

## 最终建议

本次优化的最小可靠方案是：

先做 **地图放大 + 模块三承载项目档案/累计成果 + 地图浮层承载今日状态 + 车辆详情/视频联动 + 参观模式**。

不要优先投入复杂地图边界、真实天气、截图导出、全部图表重做。这些可以后续迭代。当前最能提升客户感知和领导参观效果的，是让大屏第一眼看到全域地图运行态势，同时在左侧稳定呈现项目规模和累计成果。
